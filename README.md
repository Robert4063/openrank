# OpenPulse

## 📖 项目简介
**OpenPulse** 不仅仅是数据的展示，在这基础之上我们需要评估这个开源社区的健康程度。
在这里，你不仅可以看到一个社区的一整个生命周期，还能使用智能体预测之后的社区状态。你不仅能看到社区的健康价值，也能在这里面通过维度图找出擅长于解决问题的大牛和有潜力的新人。

## 🚀 核心功能

### 1. 社区健康评估与可视化
*   **健康指标**: 根据仓库里面每个人推送代码、comments和关注的次数（权重）来衡量每个社区的健康程度。
*   **赛博光影可视化**:
    *   **球状图**: 代表主要工作者和协作者。球的大小和工作者的贡献相关。
    *   **连线**: 代表协作者之间的关系。
        *   **<span style="color:green">绿色/相近</span>**: 关系密切并且友好。
        *   **<span style="color:red">红色/远端</span>**: 关系疏远且矛盾。
        *   **<span style="color:yellow">黄色</span>**: 中间状态。
    *   **关系判定**: 使用 comments、推送以及 NLP 情感分析（是否被接受）来衡量关系亲疏远近。

### 2. 生命周期追踪与预测
*   **历史数据**: 通过 OpenDigger 得出来的数据进行折线图绘制，呈现项目之前的样貌。
*   **趋势预测**: 使用之前的数据以及近期的趋势联合表示，预测接下来项目会发生的变化（虚线显示）。
*   **实时状态**: 向下滑动（或点击按钮）显示当前的人员关系以及社区健康程度。

### 3. 多智能体协作 (MaxKB)
*   **角色**: 实时分析当前场景，分析之前数据和预测之后数据。
*   **建议反馈**: 据此给仓库拥有者或者社区管理者相应的建议。分析和预测依据的是之前计算出来的社区关系紧密度、人员关系紧密和谐程度。

### 4. 搜索与探索
*   **热门项目**: 首页搜索框下方显示当天热度 Top 3-5 的项目（显示名称及主要人员），每日中午 12 点更新。
*   **智能搜索**: 可以使用搜索栏或 AI 智能体查询。结果首先按输入字符相似度排列，匹配度相当则按社区活跃度排列。
*   **详细视图**:
    *   **总体关系**: 呈现赛博光影球状图。
    *   **时间筛选**: 右上角可下拉显示日期，查看查询天的人员紧密关系或月份的整体贡献图。
    *   **贡献详情**: 点击球体显示截止到当天的总贡献，下方有热力图显示贡献热度和天数。

## 🧠 核心算法
*   **NLP 情感计算算法**: 用于分析互动的情感色彩（友好 vs 冲突）。
*   **传染病模型**: 用于模拟信息传播或趋势采纳。
*   **排斥力算法**: 用于关系图的视觉布局。

## 💻 用户体验流程 (UX)

### 1. 首页 (Landing Page)
*   **搜索**: 显眼的搜索框。
*   **热门推荐**: 显示当天热度 Top 3-5 项目，每日 12:00 更新。

### 2. 搜索结果
*   **排序**:
    1.  字符相似程度。
    2.  社区活跃度 (当匹配度相当时)。
*   **列表**: 结果按 Star 数从高到低排列。

### 3. 项目概况 (第一部分)
*   **可视化组合**: 折线图、直方图以及山图的结合。
    *   **背景颜色**: 代表社区总的健康程度。
    *   **折线图**: 社区每个月的紧密程度（按月计算）。
    *   **直方图**: 社区的活跃程度（按月计算）。
    *   **山图**: 项目里的持续增长。
    *   **预测**: 之后的月份用虚线显示。
*   **控制**: 右上角日期选择（年、月、日），默认显示整个项目周期。

### 4. 人员情况 (第二部分)
*   **可视化**: 贡献度从大到小排列的球体。
*   **侧边栏**: 点击 `>` 号出现表格，显示项目语言占比。
*   **智能助手**: 右下角 MaxKB 构建的多智能体。
    *   实时分析当前场景，回答用户问题。
    *   **自动 Issue**: 若问题未在 Issue 中提到，自动使用用户名字生成新 Issue，并及时提醒回复。
*   **控制**: 右上角日期选择（前三月、前两月、半年等）。

### 5. 贡献者详情
*   **进入方式**: 点击任意球体。
*   **雷达图 (左)**: 六维度展示 - 代码贡献、代码审查活跃度、Issue 沟通能力、被连接数、持久度。
*   **热力图 (右)**: 日历样式，显示贡献热度。
*   **连接**: 最下方显示该人员参与的所有项目的连接。

### 6. 对比模式
*   将两个项目的维度进行相对比。

## ⚠️ 注意事项
1.  **紧密度评价**: 使用当天以及之前的记录进行评估。
2.  **热力图**: 显示每一天的贡献程度。
3.  **数据更新**: 数据库每日中午 12 点进行抓取更新。
